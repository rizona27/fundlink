# 客户基金汇总页面改动总结 (2026-06-02)

## 概述

在 `ClientView` 页面中，点击客户姓名的渐变条展开后，在「持仓数:X支」后面增加三个小点按钮，点击后跳转到新页面 `ClientFundSummaryPage`，该页面汇总分析该客户持有的所有基金。

---

## 涉及文件

| 文件 | 操作 | 说明 |
|------|------|------|
| `lib/views/client_view.dart` | 修改 | 展开时添加三个小点按钮，导航至新页面 |
| `lib/views/client_fund_summary_view.dart` | 新建 | 客户基金汇总分析页面（4个模块） |
| `lib/services/stock_quote_service.dart` | 新建 | 腾讯行情 API 股票数据服务 |

## 一、ClientView 三个小点按钮 (`client_view.dart`)

### 位置
`itemBuilder` 中 `GradientCard` 的 `trailing` 参数，约第 614 行。

### 逻辑
- 当客户卡片展开时（`_expandedClients.contains(key)`），在原 `持仓数: X支` 后追加三个小点按钮
- 按钮样式：`CupertinoIcons.ellipsis`，16px，圆角半透明背景
- 点击后通过 `AnimationConfig.createPageRoute` 导航至 `ClientFundSummaryPage`
- 传入参数：`clientName`, `clientId`, `holdings`, `dataManager`, `fundService`

### 复刻来源
完全参照 `SummaryView` 第 1037-1069 行的模式。

## 二、ClientFundSummaryPage 四个模块

### 页面结构
- `CupertinoNavigationBar` 标题显示客户姓名，自动返回按钮
- 主体为 `ListView`，包含 4 个卡片模块
- 深浅色模式自适应

### 数据加载
- `initState` 中触发 `_loadData()`
- 并行请求所有基金的十大重仓股数据（`FundService.fetchTopHoldingsFromHtml`）
- 排序后通过 `StockQuoteService.fetchQuotes` 批量获取股票行情
- 加载中显示 `CupertinoActivityIndicator`

---

### 模块一：投资金额分布
- **类型**：`fl_chart` 饼状图（PieChart）
- **内容**：每个基金的 `totalCost` 占比
- **布局**：左侧 100x100 饼图 + 右侧图例（基金名、金额、百分比）
- 饼图空心内径 24px，段间距 2px

### 模块二：盈亏分布
- **类型**：自定义双向水平柱状图
- **设计原因**：盈亏有正有负，饼状图无法正确表达
- **实现**：绿色柱向左（亏损）、红色柱向右（盈利），中间零点线分隔
- **柱宽**：`|profit| / max(|profit|)` 按比例缩放
- **图例**：顶部显示 `← 亏损` / `盈利 →` 方向指示
- 每个条目显示：基金名、带符号金额、占总|盈亏|百分比

### 模块三：十大重仓股（按购买权重再统计）
- **算法**：`weightedRatio = Σ( 基金投入权重 × 该股票在基金中的占比 )`
  - 基金投入权重 = `fund.totalCost / totalInvestment`
- **排序**：按 `weightedRatio` 降序，取前 10
- **显示**：排名、股票名+代码、市场标签（沪A/深A/HK）、权重占比、覆盖基金数
- **行高压缩**：`vertical: 3` padding，紧凑排版
- **颜色编码**：≥5% 红色，≥2% 橙色，<2% 蓝色

### 模块四：投资方向分析
包含三个子部分：

#### 4.1 风格分布
- 六条进度条：大盘/中盘/小盘 + 价值/成长/均衡
- 数据来源：`StockQuote` 的 `marketCapStyle` 和 `valueStyle`

#### 4.2 重叠重仓股
- 列出被多支基金同时持有的股票
- 显示：股票名、代码、市场标签、覆盖基金列表、最新价/PE/PB/市值
- 最多显示 6 个

#### 4.3 投资方向总结
- 自动生成文字分析，例如：
  > 组合偏向大盘风格（大盘股占比68%），成长型股票占主导（52%），进攻性强；重点行业为电子/半导体（占比35.2%）。注意：有5只股票被多只基金重仓，存在集中风险。

## 三、StockQuoteService (`stock_quote_service.dart`)

### 数据源
腾讯行情 API：`https://qt.gtimg.cn/q={codes}`

### 字段映射（0-based 索引）
| 索引 | 字段 | 示例 |
|------|------|------|
| 3 | 最新价 | 38.20 |
| 39 | 市盈率(TTM) | 6.36 |
| 45 | 总市值(万元) | 20628944429 |
| 46 | 市净率 | 0.25 |

### StockQuote 类方法
- `marketLabel` → 沪A / 深A / HK
- `marketCapStyle` → 大盘(≥200亿) / 中盘(≥50亿) / 小盘
- `valueStyle` → 价值(低PE+低PB) / 成长(高PE或高PB) / 均衡

### 编码处理
- 优先 UTF-8（`allowMalformed: true`）
- 若解析无有效数据，回退到 Latin-1（兼容 GBK 场景）
- 字段值验证：过滤空值、`-`、`--` 等无效标记

### 市值分类回退策略
当 API 数据不可用时，根据股票代码所属板块推断：
- `6xxxxx`（沪市主板）→ 大盘
- `688xxx`（科创板）、`300/301xxx`（创业板）→ 中盘
- HK 五位数代码 → 中盘

### 批量请求
- 每批最多 20 只股票
- 内存缓存，避免重复请求

## 五、布局适配

- **宽屏（≥600px）**：模块一和二左右并排（`Row`），各占 50%
- **窄屏（<600px）**：模块一和二上下堆叠（`Column`）
- **深浅色**：背景、卡片、文字、图表颜色全部通过 `isDark` 判断
